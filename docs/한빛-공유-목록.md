## 2. 단계별 — 누가 뭘 하나

```
        ┌── 솔 메인 (콘텐츠·디자인·마케팅·기획) ──┐        ┌ 솔 메인(베타·외국인 인터뷰)
PM ─ Plan ─ Design ─ (Do) ─ Check ─ Act ─ QA ─ Report ─ Archive
                  └──── 한빛 메인 (코드) ────┘
```

- **PM·Plan·Design**: 솔이 PM처럼 주도 (지금까지)
- **Do**: 한빛이 코드 작성. 솔은 그 시간에 콘텐츠·SNS·인포그래픽 시안 병행
- **Check·Act**: 도구가 "도면대로 됐나" 자동 점검·수정 → 둘 다 결과만 검토
- **QA**: 솔이 메인 — 외국인 베타 30명 섭외·인터뷰·인포그래픽 A/B 테스트
- **Report·Archive**: 둘 다

## 3. 깃허브가 뭔가요 (왕초보 설명)

**깃허브 = 인터넷에 있는 공유 폴더 + 자동 백업.**
둘이 같은 폴더를 각자 컴퓨터에 받아 쓰고, 바뀐 걸 인터넷으로 주고받습니다.

알아둘 단어 딱 3개:
- **받기 (pull)** = 인터넷의 최신 내용을 내 컴퓨터로 가져오기
- **올리기 (commit + push)** = 내가 바꾼 걸 인터넷에 저장하기
- **충돌** = 둘이 같은 줄을 동시에 고쳤을 때 "누구 걸로 할까?" 묻는 상황

> **솔은 명령어 외울 필요 없습니다.** Claude Code에게 말로:
> "한빛 최신 거 받아줘" / "지금까지 한 거 올려줘" / "충돌 정리해줘"
> 한빛도 Codex에게 똑같이 시키면 됩니다.

## 4. 매일 작업 순서 (이대로만 하면 안전)

**① 시작할 때 — 받기**
> "한빛 최신 거 받아줘"
어제·방금 올라온 변경을 먼저 가져옵니다. (안 받고 시작하면 충돌의 원인)

**② 작업 — 평소대로**
솔은 Claude Code로, 한빛은 Codex로. 뭘 건드려도 됩니다.

**③ 한 덩어리 끝날 때마다 — 올리기**
> "지금까지 한 거 올려줘"
30분~1시간, 또는 한 가지 일 끝낼 때마다. **작게 자주**가 핵심.

**④ 충돌나면 — 같이 정리**
> "충돌 정리해줘"
도구가 양쪽 버전을 보여주고 합쳐줍니다. 무서운 거 아닙니다.

## 5. 언제 올리고 언제 받나 (타이밍 표)

| 상황 | 할 일 |
|---|---|
| 🌅 작업 시작 전 | **받기** (항상 먼저!) |
| ✏️ 한 가지 일 끝냈을 때 (30분~1h) | **올리기** |
| 🌙 하루 일 끝낼 때 | **꼭 올리기** (안 올리면 한빛이 못 봄) |
| 📣 큰 파일을 반나절+ 만질 때 | 시작 전 카톡 한마디 "나 오늘 OO 손볼게" |
| 🔔 상대가 "올렸어" 하면 | **받기** |

> 핵심 한 줄: **받고 → 작업하고 → 자주 올린다.** 이것만 지키면 사고 안 납니다.
> (깃허브는 구글닥스처럼 실시간이 아니라, "올리기" 할 때만 합쳐집니다.)
