## 협업 원칙 — 벽은 없다

솔(Claude Code)과 한빛(Codex)은 **모든 영역을 같이 작업한다.**
한빛이 짠 코드를 솔이 받아서 고쳐도 되고, 솔이 쓴 문서를 한빛이 고쳐도 된다.
"여기 손대지 마"는 없다. 단, 아래 **git 습관 3개**만 지킨다.

### 기본 담당 (벽 아님, 겹침을 줄이는 기본값일 뿐)

| 사람 | 주로 맡는 곳 | 비고 |
|---|---|---|
| 한빛 / Codex | `src/`, 앱 코드, `.codex/` | 누구나 수정 OK |
| 솔 / Claude Code | `docs/`, 콘텐츠, 영문 카피, Plan B | 누구나 수정 OK |

→ 특별한 이유가 없으면 각자 강한 쪽을 맡되, 필요하면 자유롭게 넘나든다.

### git 습관 3개 (이게 충돌을 막는 전부)

1. **시작 전 받기** — 작업 시작할 때 항상 최신부터 받는다 (pull)
2. **작게 자주 올리기** — 한 덩어리 끝날 때마다 올린다 (commit + push). 며칠 묵히지 않는다
3. **큰 파일 오래 만질 땐 한마디** — "나 오늘 OO 크게 손볼게" 카톡 한 줄

## 작업 루틴 (이 순서대로)

> 솔은 git 명령을 외울 필요 없다. Claude Code에게 말로 시키면 된다.
> 한빛도 Codex에게 똑같이 시킨다.

**1. 시작 — 받기**
> "한빛 최신 작업 받아줘" (git pull)
어제·방금 올라온 변경을 내 컴퓨터로 가져온다.

**2. 작업 — 평소대로**
한빛=Codex / 솔=Claude Code. 무엇을 건드리든 자유. 파일은 텍스트라 어느 AI든 읽고 고친다.

**3. 중간중간 — 자주 올리기 (가장 중요)**
> "지금까지 한 거 올려줘" (git commit + push)
30분~1시간 단위, 한 덩어리 끝날 때마다. 자주 올릴수록 상대가 빨리 받아 충돌이 작아진다.

**4. 충돌나면 — 같이 정리**
> "충돌 정리해줘"
둘이 같은 줄을 고치면 git이 묻는다. AI가 양쪽 버전을 보여주고 합쳐준다. 무서운 거 아니다.

> ⚠️ git은 실시간 동기화(구글닥스)가 아니다. "올리기" 할 때만 합쳐진다.
> 그래서 **받기 → 작업 → 자주 올리기**가 핵심이다.
